# Task 실패 시

> Task 실패 시 수행해야 할 롤백 및 기록 절차

## 1. 실패 인식

**다음 경우 task 실패로 간주:**
- 에러 발생하여 진행 불가
- 요구사항을 충족할 수 없음
- 예상치 못한 부작용 발견
- 사용자가 중단 요청

## 2. 롤백 절차

```
1. 변경한 파일 목록 확인
2. 가능하면 이전 상태로 복구
3. 복구 불가능하면 사용자에게 알림
4. 실패 이유와 상황 기록
```

## 3. current-state.md에 기록

```markdown
## 실패한 작업
- [ ] Task 제목
  - 실패 일시: YYYY-MM-DD HH:MM
  - 실패 이유: ...
  - 시도한 방법: ...
  - 롤백 여부: ...
```
