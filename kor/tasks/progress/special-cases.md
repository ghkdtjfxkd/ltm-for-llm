# 특수 상황 처리

> 일반적이지 않은 상황에서의 대응 방법

## 세션 중단 후 재개

```
1. current-state.md 읽기
2. "진행 중" 표시된 task 확인
3. 마지막 완료 단계 파악
4. 다음 단계부터 이어서 진행
```

## 불확실한 경우

```
- 사용자에게 질문
- 여러 옵션 제시
- 권장 방법과 이유 설명
- 결정 후 문서에 기록
```

## 충돌/모순 발견

```
1. 즉시 사용자에게 알림
2. 어떤 문서 간 충돌인지 명시
3. 가능한 해결 방법 제안
4. 사용자 결정 대기
```

## 예상치 못한 상황

- 당황하지 말고 상황 설명
- 가능한 옵션 제시
- 사용자 판단 요청
- 결정 사항 문서화
