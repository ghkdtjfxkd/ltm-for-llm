# Task 시작 전

> Task를 시작하기 전 반드시 수행해야 할 절차

## 1. 현재 상황 파악

```
1. current-state.md 읽기
2. 진행 중인 task가 있는지 확인
3. 새 task인지, 이어서 하는 task인지 판단
```

## 2. Task 분석

```
1. Task 목표가 명확한가?
2. 필요한 문서가 무엇인가? (routing 참조)
3. 예상 단계는?
4. 완료 조건은?
```

## 3. Task 분리 판단

**다음 경우 task를 작은 단위로 분리:**
- 3단계 이상의 복잡한 작업
- 여러 파일/폴더 수정 필요
- 30분 이상 걸릴 것 같은 작업
- 중간에 사용자 확인이 필요한 경우

**분리가 필요하면** → `task-splitting.md` 참조

## 체크리스트

- [ ] current-state.md 읽음
- [ ] 필요한 문서 파악 (routing 참조)
- [ ] Task 분리 필요 여부 판단
- [ ] current-state.md에 "진행 중" 표시
