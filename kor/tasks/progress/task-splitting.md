# Task 분리

> Task를 더 작은 단위로 나누는 기준과 방법

## 분리가 필요한 경우

1. **복잡도**: 5개 이상의 파일 수정
2. **시간**: 30분 이상 예상
3. **단계**: 3단계 이상의 프로세스
4. **의존성**: 다른 task의 완료가 필요

## 분리 방법

```
큰 Task: "사용자 인증 시스템 구현"
└─> 작은 Task들:
    1. 데이터베이스 스키마 설계
    2. 인증 API 엔드포인트 구현
    3. 프론트엔드 로그인 UI 구현
    4. 테스트 작성
```

## 분리 시 문서화

```markdown
## Task 분리
- 원래 Task: [큰 Task 제목]
- 분리 이유: [왜 분리했는지]
- 하위 Tasks:
  - [ ] Task 1
  - [ ] Task 2
  - [ ] Task 3
```

## 주의사항

- 각 하위 task는 독립적으로 완료 가능해야 함
- 의존성이 있다면 순서 명시
- current-state.md에 전체 구조 기록
