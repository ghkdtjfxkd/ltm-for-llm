# Memory - 지식 저장소

## 이 폴더의 역할

실제 지식과 정보를 저장하는 곳입니다.

짧고 집중적인 문서들로 나누어 필요한 것만 빠르게 찾을 수 있습니다.

## 하위 폴더들

### `architecture/`
시스템 구조, 설계 결정, 아키텍처

### `conventions/`
코딩 스타일, 네이밍 규칙, 작성 규칙

### `patterns/`
자주 쓰는 패턴, 해결책, 좋은 방법들

### `troubleshooting/`
문제 해결 방법, 에러 대응

### `domain-knowledge/`
도메인 전문 지식, 용어, 비즈니스 로직

## 핵심 원칙

- **짧게**: 한 문서는 한 주제만
- **선택적**: 모든 문서를 읽지 않고 필요한 것만
- **구체적**: 추상적 설명보다 실제 예시

## LLM 사용 지침

- **선택적 로드**: routing이나 workflow가 지시한 문서만 읽음 (전체를 읽지 않음)
- **참조 시점**: 구체적인 지식이 필요한 순간에만 해당 문서 참조
- **업데이트**: 새로운 지식을 배우면 적절한 카테고리에 문서 추가하고 routing 맵 갱신
