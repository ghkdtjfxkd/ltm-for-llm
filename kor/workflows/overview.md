# Workflows - 작업 방식

## 이 폴더의 역할

작업을 어떤 순서로, 어떻게 진행할지 정의합니다.

단순히 "무엇을"이 아니라 "어떻게, 왜, 어떤 순서로" 생각하고 일하는지 기록합니다.

## 어떤 문서가 들어갈까요?

예를 들어:
- 새 기능 개발 프로세스
- 버그 수정 절차
- 코드 리뷰 방법
- 배포 절차

## 핵심 아이디어

내가 일하는 방식을 문서로 만들어두면,
AI가 나처럼 생각하고 작업하게 됩니다.

## LLM 사용 지침

- **작업 프로세스 따르기**: 해당 작업(기능 개발, 버그 수정 등)의 워크플로우 문서를 읽고 정의된 단계를 순서대로 진행
- **사고 방식 모방**: 각 단계에서 왜 이렇게 하는지, 어떤 순서로 생각하는지 따라함
- **문서 참조**: 워크플로우가 지시하는 memory 문서들을 적절한 시점에 읽음
